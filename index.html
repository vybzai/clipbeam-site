Beam a file to the box your agent is on.

clipbeam pushes a screenshot, file, or your clipboard from your laptop onto a remote box's disk — over your existing SSH, or Tailscale — and prints the absolute path. Now the coding agent on that box can just read it.

The flow

Pair once. Then beam, and the agent reads.

Universal Clipboard stops at the SSH boundary. A remote TUI can only ingest a real on-disk file by absolute path — so clipbeam writes the file and prints the path. That's the whole point.

Set up the box

Dials in over SSH, streams the binary across, records the remote path. No open ports, no new account.

clipbeam setup user@box

Beam the file

Push a screenshot, file, or clipboard. clipbeam prints the remote absolute path on stdout.

clipbeam send screenshot.png box /root/.cache/clipbeam/in/screenshot.png

Agent reads it

On the box, clipbeam last is the path. Hand it straight to your agent.

claude "describe $(clipbeam last)"

What it does

Built for the headless-box workflow.

Your agent lives on a remote Linux box with no GUI. clipbeam is the bridge for the things it can't otherwise see.

Screenshots, instantly

clipbeam shot box grabs your screen and lands the PNG on the box's disk — so the agent can describe what you're looking at.

Any file or clipboard

clipbeam send any file, or pipe stdin. The decoded payload writes straight to disk with a path you can act on.

Agent-to-agent channel

A private inbox: clipbeam msg to send, clipbeam recv to drain it. It never touches the human's clipboard or notifications.

Prints the path, period

clipbeam last emits a bare absolute path with no trailing newline — so $(clipbeam last) drops cleanly into any command.

SSH or Tailscale

Rides the SSH tunnel you already have by default — host keys verified, no daemon. Tailscale is an option, not a requirement.

Push-only by design

Nothing leaves a machine unless you run a command. clipbeam never polls, never pulls, never loops.

For agents, not just humans

A CLI an agent can drive blind.

Every verb speaks JSON. Exit codes are a single deterministic table. And clipbeam schema describes the whole surface at runtime — so an agent can introspect and use it with zero human in the loop.

  • Self-describing. clipbeam schema emits one JSON doc: every command, its args, flags, stdout shape, and exit codes.
  • Deterministic exits. One merged exit-code table covers local, HTTP, and SSH failures — branch on it with confidence.
  • Streamable. clipbeam watch --json emits NDJSON, one line per item, for a tail-and-react loop.
  • Skill included. Ships a skill generated from clipbeam schema for Claude Code and Codex.

Security model

Rides the trust you already set up.

No new account, no inbound port, no cloud middleman. clipbeam uses the SSH you already use to reach the box.

Push-only

Nothing leaves a machine unless you act. clipbeam never polls a clipboard and can't loop with Universal Clipboard.

SSH transport by default

Rides your existing tunnel — host keys verified, no new account. Tailscale is optional, not required.

Secrets stay local

The token lives in the macOS Keychain, or a 0600 token file on Linux. Logs redact the token — it never lands in plaintext.

Bounded payloads

A hard 50 MB cap on decoded payloads, enforced before anything touches disk. Over-cap exits with code 7.
